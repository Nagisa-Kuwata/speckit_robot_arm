# 7軸ロボットアーム 3次元動作シミュレーター仕様書

## 概要
人間の片腕を模擬した、7軸ロボットアームの3次元動作を描画するシミュレーター。
ユーザーはパラメータを操作し、物理法則（動力学・運動学）に基づいた挙動を確認できる。

## 機能要件

### 1. ロボットアームの構成
- **軸数**: 7軸
- **可動制限**: 各軸に可動範囲（上限・下限）を設定する。
  - 具体的な制限値は後工程で確定。

### 2. パラメータ管理
- **初期値**:
  - ロボットは各パラメータ（軸角度など）の初期値を持つ。
  - 初期値はコード内にハードコーディングまたは設定ファイルとして事前定義される。
  - 具体的な初期値は後工程で確定。
- **更新機能**:
  - ユーザーはシミュレーション開始前および実行中にパラメータを変更可能。
  - 変更可能なパラメータには、各軸の位置（角度）、速度、加速度が含まれる。

### 3. シミュレーションと描画
- **物理モデル**:
  - 地球上の動力学（Dynamics）および運動学（Kinematics）に従う。
  - **逆運動学 (Inverse Kinematics)**:
    - ユーザーが手先の目標位置座標(x, y, z)を入力可能とする。
    - ロボットは目標位置へ到達可能な場合、制御手法を用いて軌跡を生成し、移動する（瞬間移動ではない）。
  - **数値計算手法**: ルンゲ・クッタ法を採用し、精度の高い数値積分を行う。
  - **単位系**:
    - 原則としてSI単位系（メートル、kg、秒）で統一。
    - 角度については、内部計算はラジアンで行い、表示上はラジアン/度(degree)を併記する。
- **リアルタイム性**:
  - 描画はユーザー操作や物理演算の結果を即座に反映する。
  - ロボットの状態（位置、速度、加速度など）をリアルタイムにグラフ化・描画する機能を持つ。

## 技術スタック (Constitution準拠)
- **言語**: Rust
- **ターゲット**:
  - コアロジックは組込み開発（no_std環境）を想定した設計とする。
  - 描画・UIはPC上のGUIアプリケーションとして実装する。
- **優先事項**: リアルタイム性、実行性能、演算の正確性
