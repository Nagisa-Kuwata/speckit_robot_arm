# 実装計画: SpecKit Robot Arm Simulator

## 技術的仕様

### 1. 描画・可視化
- **各軸の可視化**:
  - 各軸の位置（原点）と方向（座標軸: X, Y, Z）をシミュレーター空間上に描画する。
  - 名称ラベルを付与し、どの軸であるかを識別可能にする。
- **軌跡描画**:
  - ロボットの手先（End Effector）または指定した部位の移動軌跡を描画する。
  - **フェードアウト効果**: 一定時間が経過した軌跡は徐々に透明になり、最終的に消滅する（Trail Effect）。

### 2. 物理演算・動力学
- **摩擦モデル**:
  - **粘性摩擦**: 速度に比例する抵抗力。
  - **静止摩擦**: 動き出しに必要な最小トルク/力。
  - これらを簡易モデルとして実装し、各軸の運動方程式に組み込む。
- **重力と重力補償**:
  - 各リンクは自身の質量と重心位置に基づき、重力の影響を受ける。
  - **重力補償**: デフォルトで有効。重力による自然落下を防ぐトルクを自動的に付加する。
  - **ユーザー制御**: ユーザー入力により重力補償のON/OFFを切り替え可能とする。

### 3. パラメータ管理
- **優先順位**:
  1. ユーザーが用意したYAML設定ファイル。
  2. コード内にハードコーディングされたデフォルト値（YAMLファイルが存在しない場合）。
- **デフォルト値の参照元**:
  - **MOTOMAN-SIA30D** (Yaskawa Electric Corporation)
  - 参照URL: [MOTOMAN-SIA30D Spec](https://www.e-mechatronics.com/product/robot/assembly/lineup/sia30/spec.html)
  - 不明なパラメータ（慣性モーメント、正確な摩擦係数など）は、一般的な産業用ロボットの値を参考に推定・仮定して設定する。

## 実装ステップ

### Step 1: プロジェクト基盤構築
- Rustプロジェクトの初期化（Cargo workspace構成など検討）。
- 依存ライブラリの導入:
  - 数値計算: `nalgebra` (線形代数)
  - 描画: `bevy` または `three-d` + `egui` (GUI)
  - 設定ファイル読み込み: `serde`, `serde_yaml`

### Step 2: 物理モデルの実装 (Core Logic)
- **Kinematics (運動学)**:
  - 順運動学 (FK): 関節角度 -> 手先位置
  - 逆運動学 (IK): 手先位置 -> 関節角度 (数値解法: Jacobian inverse / pseudo-inverse等)
- **Dynamics (動力学)**:
  - Newton-Euler法 または Lagrangian法による運動方程式の実装。
  - 重力項、コリオリ・遠心力項、慣性項の計算。
  - 摩擦モデル（粘性・静止）の追加。
- **数値積分**:
  - ルンゲ・クッタ法 (RK4) の実装。

### Step 3: シミュレーター/GUIの実装
- 3D空間へのロボットモデル描画（リンク、ジョイント）。
- 座標軸、軌跡（Trail）の描画システム。
- GUIパネル実装:
  - パラメータ調整スライダー。
  - 重力補償ON/OFFスイッチ。
  - リアルタイムグラフ（位置、速度、トルク等）。

### Step 4: パラメータ設定と検証
- YAMLローダーの実装。
- MOTOMAN-SIA30Dのスペック（可搬質量、リーチ、軸構成）を反映。
- 動作検証（重力補償OFFでの落下挙動、IKによる目標追従など）。
